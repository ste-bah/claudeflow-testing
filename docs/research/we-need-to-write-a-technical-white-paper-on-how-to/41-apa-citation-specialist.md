## 1. REFERENCE LIST FORMATTING

### 1.1 General APA 7th Rules Applied

**Alphabetical Order**: All 78 sources alphabetized by first author's last name (or organization name for group authors).

**Hanging Indent**: 0.5 inches (1.27 cm) from second line onward.

**Double-Spacing**: Entire reference list double-spaced.

**DOI/URL Format**:
- DOI preferred when available: `https://doi.org/xxxxx` (no "DOI:" label)
- URL format: Full https:// URL
- No "Retrieved from" prefix (APA 7th standard)
- No retrieval date (unless content changes regularly, e.g., wikis)

**Author Format**: Last, F. M. (with period after initials)

### 1.2 Reference Types and Templates

#### Journal Articles (Not applicable - technical documentation focus)

```
Author, A. A., Author, B. B., & Author, C. C. (Year). Title of article: Subtitle.
    Title of Journal, volume(issue), pages. https://doi.org/xxxxx
```

#### AWS Documentation - Web Pages

```
Amazon Web Services. (Year). Title of page. AWS. https://aws.amazon.com/...
```

**Example**:
```
Amazon Web Services. (2025). AWS Security Hub FAQs. AWS.
    https://aws.amazon.com/security-hub/faqs/
```

#### AWS Documentation - User Guide Sections

```
Amazon Web Services. (Year). Section title. In AWS Service User Guide.
    https://docs.aws.amazon.com/...
```

**Example**:
```

    https://docs.aws.amazon.com/securityhub/latest/userguide/finding-aggregation.html
```

```
Amazon Web Services. (Year, Month Day). Blog post title. Blog Name.
    https://aws.amazon.com/blogs/...
```

**Example**:

### 1.3 Multiple Works by Same Author

When citing multiple works by the same author in the same year, use lowercase letters (a, b, c):

```
Amazon Web Services. (2025a, December). AWS Security Hub now generally available
    with near real-time analytics and risk prioritization. *AWS News Blog*.
    https://aws.amazon.com/blogs/aws/aws-security-hub-now-generally-available...

Amazon Web Services. (2025b). Central configuration in Security Hub CSPM. In *AWS
    Security Hub User Guide*.
    https://docs.aws.amazon.com/securityhub/latest/userguide/central-configuration-intro.html

    Lake. In *Amazon Security Lake User Guide*.
    https://docs.aws.amazon.com/security-lake/latest/userguide/open-cybersecurity-schema-framework.html

**In Reference List**: Alphabetize by title after year when same author/year.

**In-Text Citations**: Use letter suffix: (Amazon Web Services, 2025a), (Amazon Web Services, 2025b)

## 2. IN-TEXT CITATION FORMATTING

### 2.1 Basic Author-Date Rules

#### Narrative Citations (Author as Subject)

Use "and" between authors:

```
Amazon Web Services (2025a) announced that Security Hub now provides near real-time
analytics and risk prioritization.

Aqua Security (2025) provides Trivy as an open-source vulnerability scanner.

Smith and Jones (2020) found that automation reduces mean time to respond.

Smith, Jones, and Williams (2020) demonstrated the effectiveness of layered detection.
```

#### Parenthetical Citations (Author in Parentheses)

Use "&" between authors:

```
Security Hub 2025 introduces near real-time risk analytics (Amazon Web Services, 2025a).

Container vulnerability scanning is essential for DevSecOps (Aqua Security, 2025).

Prior research supports this approach (Smith & Jones, 2020).

The findings align with established patterns (Smith et al., 2020).
```

---

### 2.2 Multiple Authors Rules

#### Two Authors

- Narrative: Smith and Jones (2020)
- Parenthetical: (Smith & Jones, 2020)

#### Three or More Authors

Use "et al." from the FIRST citation in APA 7th:

- First citation: Smith et al. (2020)
- Subsequent: Smith et al. (2020)
- Parenthetical: (Smith et al., 2020)

---

### 2.3 Group Authors

#### First Citation

Spell out full name and establish abbreviation:

```
The American Psychological Association (APA, 2020) provides publication standards.

According to the National Institute of Standards and Technology (NIST, 2020), zero
trust architecture...

AWS Security Hub (Amazon Web Services [AWS], 2025) provides centralized visibility.
```

#### Subsequent Citations

Use abbreviation:

```
Subsequent research (APA, 2020) confirmed these guidelines.

The zero trust model (NIST, 2020) requires continuous verification.
```

---

### 2.4 Multiple Works

#### Multiple Works in One Citation

Alphabetize by first author, separate with semicolons:

```
Research supports this approach (Amazon Web Services, 2025a; Aqua Security, 2024;
InfraHouse, 2025).

Prior studies demonstrate effectiveness (Jones, 2019; Smith, 2020; Williams, 2021).
```

#### Same Author, Same Year

Use lowercase letters:

```
AWS documentation (Amazon Web Services, 2025a, 2025b, 2025c) addresses multiple
configuration scenarios.
```

---

### 2.5 Direct Quotes

Direct quotes REQUIRE page or paragraph numbers:

```
Amazon Web Services (2025a) stated that "Security Hub now provides near real-time
analytics" (para. 1).

"The delegated administrator model separates governance from operations" (Amazon
Web Services, 2025, p. 12).
```

**Note**: This white paper uses paraphrasing exclusively - no direct quotes requiring page numbers.

---

### 2.6 Secondary Sources

When citing a work you read about in another source:

```
Piaget's original work (as cited in Smith, 2020) established the foundation.
```

**Reference List**: Include only the secondary source (Smith, 2020).

---

## 3. TABLE FORMATTING

### 3.1 General Table Rules (APA 7th)

1. **Numbering**: Tables numbered consecutively (Table 1, Table 2, ...)
2. **Title**: Italicized, descriptive, title case
3. **Lines**: Horizontal lines only (top, below header, bottom)
4. **Notes**: Below table - general, specific, probability
5. **Reference in text**: "as shown in Table 1" or "(see Table 1)"

---

Row 1 data      Data            Data            Data
Row 2 data      Data            Data            Data
Row 3 data      Data            Data            Data
----------------------------------------------------------------

Note. General note explaining table. N = sample size.
*Specific note. **p < .01.
```
